# What Makes Yantra Feel Jarvis Level

The build plan uses the phrase "Jarvis-level" to describe the target experience. That does not mean a flashy assistant voice or a generic all-purpose chatbot. It means the AI should feel deeply integrated, context-aware, and useful across the learner's journey.

Yantra feels advanced when several things work together:

- it knows the platform and uses Yantra-specific language
- it knows the learner's level, goals, and progress
- it retrieves the right knowledge instead of guessing
- it gives room-specific feedback instead of generic advice
- it remembers prior work and adjusts future teaching

A normal chatbot mostly answers the latest message. Yantra is supposed to answer the message in context:

- what room the learner is in
- what concept they were studying
- what mistakes they made recently
- what the most useful next action is

The build plan also treats tool use as part of the experience. A Jarvis-level teacher can decide when a quiz, diagram, code example, search, or debugging hint would help. The learner should not have to micromanage those switches.

For the current local service, only a small part of that vision exists:

- local retrieval over Yantra docs
- student context fields in the prompt
- terminal and API chat testing
- model-backed responses through Copilot CLI

That is still useful. It proves the teaching loop before expensive infrastructure is added. Yantra should describe the current state honestly while keeping the long-term vision clear.
