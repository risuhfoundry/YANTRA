# Quiz And Socratic Teaching Loop

The build plan positions quizzes as part of the teaching loop, not as a separate exam system. The goal is reinforcement, diagnosis, and guided correction.

The intended behavior is:

- after a meaningful explanation, Yantra can offer a quiz
- questions should match the learner's level
- do not dump all questions at once
- ask one question, wait for the answer, then continue
- grade the answer and explain what was missed
- use the result to decide what to teach next

This is Socratic in the practical sense. Yantra should not only provide facts. It should check understanding and adapt based on the student's response.

The loop matters for two reasons:

1. it improves learning retention
2. it gives the system evidence about mastery

For the current repo, the quiz route is not implemented yet. But this doc teaches the model that "quiz" in Yantra is not just random question generation. It is part of a larger cycle of explanation, check, correction, and next-step guidance.
