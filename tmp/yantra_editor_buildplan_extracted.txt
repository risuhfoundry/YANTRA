01 / PRODUCT CONTEXT
Why the Editor Belongs in Yantra
Yantra is an AI-native learning OS — skill diagnosis → personalized roadmaps → certifications → job matching. The
Python Room is already live and documented. Learners are already expected to write and run code inside the platform.
The /editor page is the natural execution surface for that promise — not a bolt-on feature, but the place where
roadmap-guided learning becomes hands-on practice.
The editor must feel like a native part of Yantra — same #060608 background, same glass card aesthetic, same auth
layer, same AI assistant — not a third-party iframe dropped into the page.

06 / BACKEND WORK — TOBE
● DAY 1 — Execution Engine
Tobe (Backend)
■ Create app/api/editor/run/route.ts
– This is a Next.js API route. Extract { code, language, stdin } from the request body.


--- PAGE 4 ---
4
– Validate the Supabase session server-side before executing — reject unauthenticated requests with 401.
■ Piston API integration
– Endpoint: POST https://emkc.org/api/v2/piston/execute
– Body: { language, version: '*', files: [{ content: code }], stdin }
– Piston returns: { run: { stdout, stderr, code, cpu_time, wall_time } } — map this to the agreed response shape.
– For v1 use the public Piston instance (free, no key needed). Document self-hosting steps in the repo for later.
■ 10-second execution timeout
– Wrap the Piston fetch in an AbortController with a 10 000 ms timeout. If it fires, return a timeout error in stderr.
■ Rate limiting
– Read the user's sub (user ID) from the Supabase session.
– Use an in-memory store (or Upstash Redis if available) to cap at 30 runs per user per minute.
– Return HTTP 429 with a clear message if limit is hit — Rishi will display this in the output panel.
■ Language version pinning
– Call GET https://emkc.org/api/v2/piston/runtimes once and cache the latest versions for Python, JavaScript,
TypeScript.
– Hardcode these versions in a SUPPORTED_LANGUAGES config object to avoid mismatch errors.
● DAY 2 — AI Endpoint & Database
Tobe (Backend)
■ Create app/api/editor/ai/route.ts
– Copy the Gemini setup pattern from app/api/chat/route.ts — reuse the same client init and streaming logic.
– System prompt: 'You are Yantra's learning assistant. Explain the following {language} code clearly. The learner's
current skill level is {level} from their profile. Keep the explanation practical and learning-focused.'
– Stream the response back using ReadableStream + TransformStream so Rishi can render tokens progressively.
■ Create Supabase table: code_snippets
– Mirror the chat_histories table pattern exactly.
id          uuid primary key default gen_random_uuid()
user_id     uuid references auth.users(id) on delete cascade
language    text not null
title       text
code        text not null
created_at  timestamptz default now()
updated_at  timestamptz default now()
– Add RLS policy: users can only read/write their own snippets.
– Add index on user_id for fast lookup.
■ Create app/api/editor/save/route.ts
– POST endpoint. Validate session, upsert into code_snippets by user_id + title. Return the saved row ID.
■ Create app/api/editor/snippets/route.ts
– GET endpoint. Returns the last 20 snippets for the authenticated user ordered by updated_at desc.
– Rishi displays these in the left sidebar as snippet history.

09 / FUTURE UPGRADES (POST v1)
Where to take the editor after launch
■ Roadmap-aware AI: Pull the learner's current Yantra skill level and roadmap position into the Gemini prompt. The
explanation depth should match where the learner is — not generic.
■ Multi-file tabs: Add a tab strip above the editor. Store tabs in local state; persist to Supabase as a snippet group.
■ Real terminal via WebSockets: Replace the stdout panel with an xterm.js terminal connected to a Docker
container. This enables npm install, file navigation, persistent processes.
■ LSP support: Add the Monaco language client package for JavaScript/TypeScript LSP. Enables go-to-definition,
real IntelliSense, inline error squiggles.
■ Self-hosted Piston: Deploy Piston on a VPS or Railway for zero cold-start, custom resource limits, and private
language environments.
■ Snippet library page: A dedicated /editor/snippets route showing all saved snippets with search, tags, and
one-click load.
